# Harness Engineering Research - 2026-06-24

This note records the research basis for the initial harness-pulmuone structure. It is intentionally short so Claude Code can reload it without wasting context.

## Findings

1. Harness engineering is less about a large prompt and more about repository-local constraints, feedback loops, quality gates, and state that the agent can inspect.
2. A short root instruction file plus deeper structured docs is preferred over one long manual. The root file should act as a map.
3. Plans should be first-class artifacts. Complex work needs checked-in plans, progress notes, and decision history.
4. Agent-legible architecture matters. Code, docs, schemas, tests, and validation commands should live in the repo rather than in external chat history.
5. Claude Code now treats `CLAUDE.md`, `.claude/skills`, commands, hooks, and settings as the natural extension layer. This project uses `CLAUDE.md`, one planning skill, and minimal shared hooks for secret protection and formatting.
6. Supabase SSR for Next.js should use `@supabase/ssr` with cookie-aware server and Proxy clients.
7. New Supabase projects should prefer `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` over legacy anon keys. Secret keys must stay server-only and should never be committed.
8. Next.js recommends Vitest and React Testing Library for focused unit tests. Async Server Components may need broader integration or e2e tests only when the risk justifies it.
9. Vercel already provides the deployment path for Next.js projects, so this repo does not add separate GitHub Actions CI/CD by default.

## Design Decisions For This Repo

- Keep the harness Claude Code specific.
- Use `CLAUDE.md` as the entry map and `.claude/skills/plan-development` as the planning trigger.
- Require PRD, FRD, and TRD before plan generation.
- Store plans in `docs/plans`, stories in `docs/stories`, and live progress in `STATUS.md`.
- Use `pnpm validate:quick` for the inner loop and `pnpm validate` before completion.
- Provide Supabase wiring that builds without credentials but clearly reports pending configuration.
